## 3. 🔴 전이 게이트 FAIL → 진단으로 대체 (이 세션의 핵심 판단)

10스텝 게이트 실측: ce 10.92(step1) → 8.55(step2-10 평균), 평균 ≈9.7 ≫ 3.0 →
**공식 FAIL**(러너 자동 중단 — 과금 보호 정상 작동). 그러나 원인 분리 진단
(train[:48], tta1, stage2 off, 어댑터 직접 로드)이 다른 그림을 보여줌:

| 변형 | EM | pairwise | 해석 |
|---|---|---|---|
| A 비프루닝 1.1MP | 0.5625 | 0.8924 | **전이 실재**(랜덤 pairwise 0.5) |
| A2 비프루닝 602112px | 0.5625 | 0.8924 | 해상도 이동 영향 **0** (게이트 문서의 1순위 용의자 기각) |
| B 프루닝 · motion 0 | 0.5417 | 0.8750 | 프루닝 충격 −2pp뿐 |
| C 프루닝 · motion 0.3 | 0.5417 | 0.8681 | **모션 keep-set은 기존과 동급**(EM 동일) |

- ce≈10은 "무전이"가 아니라 **프롬프트 이동 하의 확신-오보정**(CE는 보정 민감,
  EM/쌍순서는 불변량 — body의 순서판별 능력은 대부분 통과했다).
- 같은 어댑터·같은 코드·같은 해상도의 구 박스(07-20~21)는 이 warm-start로
  1500스텝을 완주해 test ≈0.90급 모델을 냈다(포스트모템 실측). 산술적으로 구
  박스의 게이트도 같은 ce를 봤어야 하므로, **ce<3.0 임계 자체가 이 셋업에
  미보정**이었다고 판단(게이트 설계 시점엔 "warm-start면 시작부터 유능"을
  가정했으나, 그 가정은 Ver8→Ver11 프롬프트 이동 때문에 성립하지 않음).
- 어댑터 파일 손상(텐서 통계)·허브 스냅샷 변경·코드북 불일치(Ver8/Ver11 perm
  대조)·모션 코드 버그(C≈B) 전부 개별 기각.

**결정: `SNUAI_SKIP_GATE=1`로 본런 진행**(레시피 무변경) + 보호장치를 게이트보다
엄격한 위치로 이동 — **step 200(≈1.8h)에서 windowed ce ≥ 3.0이면 즉시 kill**
(워처 자동 알람). 근거: 게이트의 목적(망한 전제에 13.5h를 태우는 것 방지)은 위
직접 측정으로 이미 달성됐고, 남은 리스크는 "적응 속도"뿐이라 학습 초반 곡선이
정확한 감시 지점이다.
