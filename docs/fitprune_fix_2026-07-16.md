# FitPrune stuff-over-things 수정 기록 (2026-07-16)

커밋: `0ee8e33` (객체성 블렌드 + MMR 선택), 선행 `24da29e` (모달리티별 센터링).
검증: `tests/test_fitprune.py` 63개 통과 + `runs/prune_viz` 동일 샘플 3건 전/후 재렌더.

## 1. 증상 — 무엇이 잘못돼 있었나

`runs/prune_viz` 실측(train 샘플 3건: diZi5g 스키, u7w0lr 수영장, 2vqGOF 렌즈)에서
50% 프루닝의 keep-set이 일관되게 다음 패턴을 보였다:

- **살아남는 것**: 물 표면, 눈밭, 하늘, 벽 타일 — 캡션의 장면 명사(pool/ice/snow)와
  매칭되는 대면적 균질 텍스처("stuff").
- **잘리는 것**: 스키어, 공, 흰 수영모, 얼굴, 노란 렌즈 — 시간 순서를 실제로 판별하는
  소형 전경 물체("things"). 스키 샘플에서는 E4("a skier moves forward and jumps onto
  a rail")의 유일한 단서인 스키어 몸통이 img3/img4에서 거의 전부 프루닝됐다.

즉 프루닝이 "순서 추론에 필요한 재료"를 정확히 골라서 버리고 있었다.

## 2. 원인 — 왜 그렇게 됐나 (메커니즘 3개)

1. **장면 명사의 예산 독식**: text_pool=max라서 물 패치 수백 개가 각각 "pool" 앵커와
   높은 코사인을 얻는다. top-k 선택은 중복 개념이 없으므로 서로 거의 동일한 배경
   패치들이 예산의 대부분을 채운다(diversity-fill은 20%뿐). 장면 명사는 4프레임
   공통이라 순서 신호가 0인데도 점수가 가장 높다.
2. **소형 물체의 2×2 머지 희석**: 공·모자·렌즈는 머지 셀 하나보다 작아 배경과
   평균되면서 앵커와의 코사인이 낮아진다.
3. **센터링의 부작용 — 객체성 신호 소거**: `_center_unit`은 residual을
   unit-normalize하므로 "이미지 평균에서 얼마나 벗어났는가"라는 크기 정보(전경일수록
   큼)를 버린다. 배경 패치는 residual이 작아도 정규화 후 방향 노이즈가 증폭돼
   우연히 앵커와 정렬될 수 있다.

참고: 같은 날 오전의 센터링 수정(`24da29e`, rank-1 퇴화 해소)은 "4개 이벤트 맵이
전부 동일해지는" 문제를 고친 것이고, 이 문서의 수정은 그 위에서 드러난 별개의
실패 모드다(이벤트 맵은 구분되는데, 어느 맵에서든 배경이 이긴다).

## 3. 수정 — 무엇을 어떻게 바꿨나

`src/snuai11/fitprune.py`, 두 항 모두 기본 on, 각각 0으로 두면 이전 파이프라인을
**정확히**(keep-set 비트 단위 동일) 재현 — `test_legacy_path_via_zero_flags_matches_old_selection`으로 고정.

### ① 객체성 블렌드 — `objectness_weight` (기본 0.3)

```
score = (1 − w) · minmax(cos) + w · minmax(‖v − centroid‖)
```

- **무엇**: 각 토큰의 센트로이드-residual **norm**(= 정규화가 버리던 그 크기)을
  per-image min-max 후 코사인 점수와 가중합.
- **왜 이 형태**: 원인 3을 정면으로 고친다 — 전경 물체는 이미지 평균에서 멀고
  균질 배경은 가깝다. min-max를 먼저 하는 이유는 두 항을 같은 [0,1] 스케일에
  올려 w가 실제 혼합비가 되게 하기 위함이고, min-max는 단조변환이라 w=0이면
  코사인 랭킹이 완전히 보존된다(ablation이 정확해짐).
- **왜 0.3**: 객체성은 캡션과 무관한 prior라 주가 되면 안 되고(캡션 조건부
  선택이라는 설계 정체성 유지), 소수 지분이어도 아래 MMR과 결합하면 구제가
  보장된다(④ 참조). 튜닝값이 아니라 구조적 선택 — Ver11은 로컬 홀드아웃이 없어
  숫자 튜닝 근거를 만들 수단이 LB 슬롯뿐이므로, 유도 가능한 범위에서 보수적으로
  고정하고 S3 paired A/B로 사후 검증하는 방침.

### ② MMR 선택 — `mmr_lambda` (기본 0.5)

```
반복: argmax( score_i − λ · relu(max_j∈kept cos(v_i, v_j)) )
```

- **무엇**: top-k(80%) + diversity-fill(20%) 2단 선택을 예산 전체에 대한 greedy
  maximal-marginal-relevance 1패스로 교체. 유사도는 센터링 공간에서 계산.
- **왜 이 형태**: 원인 1을 정면으로 고친다 — 서로 거의 동일한 배경 패치는 첫
  대표가 뽑히는 순간 나머지 전부가 λ만큼 감점돼 소수 대표로 압축되고, 남는
  예산이 점수는 낮지만 새로운 내용(전경 물체)으로 흐른다. relu를 씌운 이유는
  "비유사함"에 보상을 주지 않기 위해서다(그건 구 diversity-fill의 outlier 추종
  행동인데, MMR이 그 역할을 흡수하므로 중복 페널티만 남긴다). mmr_lambda > 0이면
  diversity_frac은 무시된다.
- **왜 0.5 (유도값)**: 두 항이 [0,1] 스케일이므로 최악 케이스 — 전경이 코사인
  최저(blend 점수 = w), 배경이 완전 중복이면서 코사인 최고(dedup 후 점수 =
  (1−w) − λ) — 에서 전경 구제가 보장되려면 **λ > 1 − 2w = 0.4**. 0.5는 클러스터
  내 유사도가 1에 못 미치는 실측 여유분. 실제로 처음 λ=0.3으로 구현했을 때
  합성 테스트(`test_objectness_blend_rescues_pruned_foreground`)에서 배경 dedup
  후 점수(0.36+)가 전경(0.30)을 여전히 이겨 실패했고, 이 유도로 0.5로 올렸다.
- **비용**: 예산 k회 반복 × O(N) — 기존 diversity 루프와 같은 복잡도, 샘플당
  수 ms 수준.

### 배선

- 플래그: `--objectness-weight`, `--mmr-lambda`를 `train_sft.py` / `infer.py` /
  `visualize_prune.py`에 추가. `run_a100.sh ws8` 레시피는 기본값을 그대로 상속.
- 시각화: 이미지마다 OBJ(객체성 맵) 패널 추가 — 블렌드 입력을 눈으로 검증 가능.

## 4. 검증 결과

- **CPU 테스트 63개 통과.** 신규: 객체성 랭킹, 전경 구제 회귀(픽스처는 시각
  센터링이 공유 방향을 지우는 것까지 반영해 배경 2클러스터 + 전경 2토큰으로 구성
  — 단일 공유 방향 픽스처는 센터링에 지워져 재현이 안 된다), MMR 중복 압축,
  결정성, 예산/정렬 불변량, legacy 경로 일치.
- **동일 샘플 3건 재렌더 (22:52)**: 스키어 머리·상체, 낚시 구멍, 바위 위 그룹 +
  빨간 보트, 물속 수영 선수, 눈+손+렌즈, 테이블의 여성 — 이전에 잘리던 단서가
  전부 keep-set에 들어옴. 물·눈·벽은 소수 대표만 남음(MMR 압축 확인).
- **유보**: OBJ 패널만으로는 객체성 항의 기여도가 시각적으로 불명확한 컷이 있음
  (개선의 상당 부분이 MMR 쪽일 수 있음) → 항별 기여는 S3 ablation의 몫.

## 5. 제약

- **keep-set이 바뀌므로 이 시점 이전 프루닝 선택과의 A/B 비교 금지** (시간·VRAM
  측정치는 유효, parity 게이트는 프루닝 선택과 무관하므로 영향 없음).
- SFT는 프루닝 켠 분포(prune_prob 0.75)에 적응하므로, **본런 이후 keep-set을 또
  바꾸면 학습/추론 불일치** — 이번 수정이 본런 전 마지막 keep-set 변경이어야 한다.

## 6. 남은 개선 여지 (우선순위순)

1. **S3 paired A/B가 최우선** — 위 전부는 정성 판단이다. train 서브셋 500~1,000건에
   대해 `--no-prune`(상한) / 현행 / `--objectness-weight 0 --mmr-lambda 0`(구 선택) /
   w-only / λ-only 를 같은 어댑터로 paired 채점. no-prune과 현행의 격차가 ≈0이면
   프루닝 품질 개선 계열 전체 종결, 격차가 있으면 항별 기여와 w·λ 재조정 근거 확보.
   A100 본런 후 같은 박스에서 추론만 돌리면 된다(~1시간대).
2. **객체성 항의 저수준 통계 오염 점검**: residual norm은 밝기·텍스처 주파수 같은
   저수준 통계와 상관될 수 있다(OBJ 패널이 애매했던 이유 후보). 개선안: (a) minmax
   대신 rank-normalize(outlier 강건), (b) mean 대신 median 센트로이드(전경이 평균을
   끌어당기는 것에 강건), (c) 상위 주성분 1~2개 제거 후 norm(whitening 근사).
   각각 순수 텐서 변경이라 CPU 테스트로 안전망 가능 — 단 keep-set이 또 바뀌므로
   본런 전에 할 게 아니면 S3 격차 확인 후에만.
3. **MMR 유사도에 공간 좌표 결합**: 현재 dedup은 임베딩 공간에서만 — 공간적으로
   멀리 떨어진 유사 텍스처도 같이 눌린다. `sim' = α·cos + (1−α)·spatial_proximity`로
   바꾸면 "같은 물이라도 다른 위치면 덜 중복" 시맨틱이 되고 공간 커버리지가
   부드럽게 보장된다. 효과 불확실 — S3에서 현행이 이미 충분하면 불필요.
4. **캐스케이드 escalation률을 무료 신호로 활용**: stage-1 마진이 τ 미만이면
   풀토큰 재검하는 구조이므로, test 추론 로그의 escalation률이 구 선택 대비
   떨어지면 "stage-1이 더 자신 있어졌다"는 간접 증거다(라벨 불필요, 규정 안전).
   run 로그에 이미 기록되므로 추가 구현 0.
5. **per-image 적응형 keep_ratio**: 점수 분포가 평평한(정보가 고른) 이미지는 더
   keep — 지금은 일률 50%. VRAM 예산 관리가 복잡해져(배치 내 가변 길이) 이득
   확인 전엔 비추천.
6. **프레임 간 대비 점수**(같은 이벤트에 대한 다른 3장의 매칭을 빼서 프레임 고유
   신호만 남기기): 이론상 가장 태스크 정합적이나 `keep_indices_for_image`를 4장
   동시 인터페이스로 바꾸는 공사 + "4이벤트 분해가 파편"이라는 S1 확인 사항이
   여기에도 부분 적용됨. ①~③으로 부족할 때만.
