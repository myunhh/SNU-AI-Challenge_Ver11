## 0. 요약 (TL;DR)

- key 대비 **59/819 오답** (key일치 760). 하지만 **실제 LB는 0.911로 SFT-1600(0.902)보다 높다.**
- **key일치로는 dpo600(760) < sft1600(762)인데 실제 LB는 dpo600 > sft1600.**
  → **이 key는 dpo vs sft를 잘못 서열화한다. 이 key로 DPO 체크포인트를 채점하면 오도된다.**
  "DPO가 망친 17개" 중 상당수는 실제로는 DPO가 고친 것(key가 틀림)일 수 있다.
- 오류의 **80%가 국소 스왑**(kendall d=1이 35, d=2가 12). 완전역전 1, 과반역전 5개뿐.
  대부분은 **거의 동일한 인접 프레임의 코인플립** — 정보량이 적고 key 자체도 가장 불안정한 구간.
- 진짜 고칠 값어치 있는 모델 약점은 소수의 명확한 클러스터에 몰려 있음(§3).
- 오답 59개 중 최소 4~6개는 **모델이 아니라 key가 틀린/논쟁적** 항목(§4).

## 2. DPO 신규 오답 17개의 성격 (SFT→DPO 퇴행)

15/17이 d=1~2 (단일 국소 스왑). 즉 **DPO는 SFT가 맞힌 답에서 딱 한 쌍의 얇은-마진 인접쌍을 뒤집는다.**
근본 원인 분류:
- (a) near-duplicate 인접 프레임 코인플립 — 다수 (0AezL1, 151Vxf, 706QNR, qkSjok, vIO9v9, xhhhpg, vh0wEB, ynzxgr …)
- (b) establishing-shot(넓은/풍경 컷)을 앞에 두는 prior — eEcInl, 706QNR
- (c) 캡션 절 순서를 시각 상태보다 우선(literalism) — NkMj3t, CbgVj3, Kk83WO
- (d) 초기상태 vs 사건결과 혼동 — CbgVj3(등돌리기 전/후), S0cFZA(넣기 vs 꺼내기)
- (e) **화면 위 텍스트가 곧 정답인데 무시** — aPKUZi (수영 텔레메트리 pace/stroke가 단조증가, 정답이 1-2-3-4로 화면에 적혀 있음. 모델은 미세 텍스트를 못 읽고 두 'easy' 프레임을 코인플립)

트라젝토리 관찰: 대부분(47/59) dpo200/400/600/800/1000이 **똑같이** 틀림(`====`) → 학습 초반에 굳어진 편향. 단 xhhhpg는 `X===`(200은 맞다가 이후 퇴행), 0AezL1·NkMj3t·ynzxgr는 `=X==`(400만 맞음) → DPO가 학습 중 정답에서 멀어진 drift 사례.

## 3. 진짜 고칠 값어치 있는 모델 약점 (clean, 우선순위 순)

### A. 시간의 화살(방향) — 단일 피사체 단조운동 클립 ★최우선
- **lmOtxf (튜브 폭포)**: 모델이 **폭포를 거꾸로(아래→위) 배열**. 물은 중력으로 내려간다는 물리가 정답을 강제하는데 정확히 반대. 코인플립도 key오류도 아닌 **명백한 물리-인과 방향 실패.**
- **yx3vew / 8L7TfG / b44PjE (구름사다리)**: 좌우 횡단 방향을 신뢰성 있게 못 정함(yx3vew는 완전역전). 놀이터 bar 도메인 반복 취약.
- **FOraJM (달리는 여성)**: 트레일링 샷에서 접근/후퇴(단조 거리변화) 방향 역전.
→ 랜드마크가 적고 한 피사체가 단조롭게 움직이는 클립에서 **arrow-of-time을 못 고정하고 뒤집는다.** 물리(중력/흐름)·거리단조성으로 정답이 정해지는 케이스가 많아 **학습 신호가 깨끗하고 효과가 클 구간.**

### B. 장면/피사체 그룹핑 — 2-클립 합성 컷
- **hHJkXc**: 빨강 체조선수 3프레임 뒤에 분홍 1프레임이어야 하는데 빨강 프레임 하나를 분홍 뒤로 보냄(그룹 깨짐).
- dpo쪽 **CbgVj3, NkMj3t, 0AezL1**도 동일 — 같은 장면 프레임을 연속으로 유지하지 못함.
→ "같은 씬끼리 붙인다"는 제약을 명시 학습하면 개선 여지.

### C. 화면 텍스트 그라운딩

### D. 초기상태 vs 결과 / 캡션 literalism (소수, 중간 우선)

## 5. 모델 개선 액션 (DPO 관점)

1. **평가를 실제 LB로.** 이 self-referential key는 DPO/SFT 서열을 뒤집으므로 체크포인트 선택 기준으로 부적합. 최소한 하드 73개는 채점에서 제외하거나 재검증.
2. **선호쌍 정제**:
   - near-duplicate 코인플립(시각적으로 거의 동일한 d=1 쌍)은 **선호 신호에서 제거/다운웨이트** — 노이즈만 주입.
   - §4의 key오류 후보를 수정하거나 제외.
   - 선호쌍을 **마진 임계값 이상**만 사용(얇은-마진 플립이 17개 신규오답의 정체).
3. **타깃 신호 추가**(정답이 명확한 구간에 집중):
   - 방향/물리: 폭포·유수(하류), 접근/후퇴 단조거리, 놀이터 bar 횡단.
   - 2-클립 그룹핑: "같은 씬 연속" 제약.
   - 화면 텍스트(스코어/텔레메트리) OCR 그라운딩.
4. **DPO 과도-플립 억제**: SFT가 확신-정답인 예를 뒤집지 않도록 KL 페널티↑ 또는 학습률↓ 검토(17개 신규오답 대부분이 얇은-마진 강제플립).
5. CoT에 "물이 어느 방향으로 흐르나 / 같은 장소 프레임인가 / 화면에 순서를 알려주는 숫자·마커가 있나"를 명시 자문 단계로 추가.
